ES6 features:
1. Apply.js, 1-7 lines - RankClass is defined as a class because it represents a specific entity with properties (college, major, rank) and a constructor to initialize those properties. Each instance of RankClass represents a major choice with its associated college, major, and rank. It is used to update the chosen-majors-table in the HTML document.
2. Apply.js, 32-36 line - the variable "specialNum" is declared as a new Map using the Map constructor function. We need special treatment for some ordinal words, such as first, second, and third.
3. Apply.js, 13 line - the variable "i" is declared using the let keyword in the for loop because we want to limit the scope of the variable "i" to the for loop block.
4. Apply.js, 14-15 lines - the variables "col" and "tit" are declared using the const keyword to reference elements because we don't want their values to be re-assigned.
5. Apply.js, 63-64 lines - the variables "total" and "choices" are declared using let because we want to be able to modify their values later in the code to show the Total Number of Majors Applied, and the variable "choices" in this code is an array that is used to store the input values of the user's choices in ranking majors to later update the table.
6. Apply.js, 72-79 lines - variables "table", "collegeCell", "majorCell", "rankCell" are declared using const because we don't want to modify already updated information in table.
7. Program.js, 1-10 lines - Variables "messages", "randomIndex", "randomMessage", "promotionBlock" are defined using const because they are not meant to be reassigned to new values.
The messages variable is an array of strings that contains different promotional messages. Since this array does not need to be modified, it is defined using const.
The randomIndex variable is used to store a randomly generated index to access a random message from the messages array. This value does not need to be changed after it is initially assigned, so it is defined using const.
The randomMessage variable is used to store the randomly selected message from the messages array. This value does not need to be changed after it is initially assigned, so it is defined using const.
The promotionBlock variable is used to store the HTML element with the ID "randomMessage". This element is not expected to change after it is initially assigned, so it is defined using const.
